*   循环条件：

    *   相向指针：`left < right`（找一对值）/ `left <= right`（找单个值 / 处理奇数长度）。
    *   滑动窗口：外层`right`遍历，内层`while`收缩`left`（避免嵌套 for 循环）。
    *   快慢指针：链表需判断`fast != null && fast.next != null`（避免空指针）。

*   边界处理：

    *   数组 / 字符串先判断`null`或空（`nums.length == 0`）。
    *   滑动窗口结果需兜底（比如`minLen`初始为`Integer.MAX_VALUE`，最后判断是否更新）。

*   时间复杂度：所有双指针模板都是 O (n)，比嵌套循环 O (n²) 高效。

1.  相向双指针：适用于有序数组 / 字符串的两端逼近问题，核心是「根据条件移动左右指针」。
2.  同向双指针（滑动窗口）：适用于子数组 / 子串的区间问题，核心是「右指针扩张、左指针收缩」。
3.  快慢指针：适用于链表 / 数组的遍历去重 / 找环问题，核心是「快指针遍历、慢指针记录有效位置」。

这三类模板覆盖了面试中所有双指针高频题，记住模板 + 刷 2-3 道对应经典题（LeetCode 167、209、141、26），就能熟练掌握。
