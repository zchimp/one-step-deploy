### 关键技巧（面试避坑）

&#x20;

1.  访问标记的使用：

    *   树：无需标记（无环），除非有父节点重复访问的场景。
    *   图 / 矩阵：必须标记（有环），用`boolean[][]`（矩阵）/`Set`（图）/`int[]`（节点编号）。
2.  DFS 回溯的关键：

    *   找「所有解」（比如所有路径）必须恢复状态（`path.remove()`+`visited恢复`）。
    *   找「是否存在」「数量」无需回溯（比如岛屿数量）。
3.  BFS 层序的关键：

    *   必须用`int size = queue.size()`记录当前层节点数，否则无法区分层数（步数）。
    *   最短路径问题：起始节点入队时步数为 0，每遍历一层步数 + 1。
4.  矩阵方向处理：

    *   定义方向数组`int[][] dirs = {{-1,0},{1,0},{0,-1},{0,1}}`，避免重复写上下左右逻辑。

&#x20;

&#x20;

### 总结

&#x20;

1.  DFS 模板：核心是「递归终止条件→标记访问→遍历邻接节点→回溯（可选）」，适合找所有解、路径。
2.  BFS 模板：核心是「队列初始化→层序遍历（size 控制层数）→邻接节点入队→标记访问」，适合找最短路径、层序遍历。
3.  场景适配：树的 DFS/BFS 无需访问标记，图 / 矩阵必须加；找最短路径用 BFS，找所有路径用 DFS。
