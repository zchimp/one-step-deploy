### 核心解题步骤（DP 的灵魂，先记死）

不管什么 DP 题，都按这 5 步走，不会跑偏：

1.  状态定义：明确`dp[i][j]`/`dp[i]`代表什么（比如`dp[i]`表示前 i 个元素的最大和）。
2.  状态转移方程：核心！找到`dp[i]`和`dp[i-1]`/`dp[i][j-1]`的关系。
3.  初始化：确定边界值（比如`dp[0]`/`dp[0][j]`的初始值）。
4.  遍历顺序：确定是正序、倒序、还是斜序遍历（比如背包 DP 需倒序）。
5.  返回结果：明确最终要返回`dp[n]`/`dp[n][m]`还是其他值。



关键技巧（面试避坑 + 提分）
空间优化：
一维 DP 能解决的问题，绝不使用二维（比如爬楼梯用两个变量代替数组）。
二维 DP（比如背包）可优化为一维，核心是「容量倒序遍历」（避免重复选）。
初始化细节：
最值问题：初始化为Integer.MIN\_VALUE/0（根据场景）。
计数问题：初始化dp\[0] = 1（空集 / 初始状态的方案数为 1）。
遍历顺序：
线性 DP：正序（依赖前一个状态）。
0-1 背包：物品正序、容量倒序（避免重复选）。
完全背包：物品正序、容量正序（可重复选）。
区间 DP：斜序（从短区间到长区间）。
状态转移推导：
从「最后一步」推导（比如爬楼梯最后一步是走 1 阶或 2 阶）。
分类讨论（比如选 / 不选当前物品、字符相等 / 不相等）。
总结
DP 模板核心：「状态定义→初始化→遍历顺序→状态转移→返回结果」五步，其中状态转移方程是核心中的核心。
场景适配：
线性问题（最大子数组和、爬楼梯）：一维 DP，正序遍历。
背包问题（0-1 / 完全）：一维 DP，注意容量遍历顺序。
字符串匹配（LCS / 编辑距离）：二维 DP，正序遍历。
区间问题（回文子串）：二维 DP，斜序遍历。
