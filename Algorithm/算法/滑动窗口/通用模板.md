滑动窗口本质是同向双指针，通过「右指针扩张窗口」、「左指针收缩窗口」，在 O (n) 时间复杂度内解决子数组 / 子字符串的区间问题，核心是：

&#x20;

1.  明确「窗口内要维护的条件」（比如和≥target、包含所有目标字符、无重复字符）。
2.  明确「何时扩张右指针」（无脑遍历）、「何时收缩左指针」（窗口不满足条件 / 满足条件后优化）。
3.  明确「何时更新结果」（收缩时 / 扩张时）。

```java
/**
 * 滑动窗口通用模板（适配绝大多数场景）
 * @param s/ nums  输入的字符串/数组
 * @param k/target 目标条件（长度/和/字符集等）
 * @return 符合条件的结果（长度/数量/子串等）
 */
public int slidingWindowTemplate(String s, int k) {
    // 1. 初始化工具（根据场景选择：哈希表/数组/计数器/窗口和）
    HashMap<Character, Integer> window = new HashMap<>(); // 字符计数（字符串场景）
    // int[] window = new int[26]; // 字母场景更高效
    // int windowSum = 0; // 数组和场景
    
    // 2. 初始化指针和结果
    int left = 0, right = 0; // 左右指针，窗口区间 [left, right) 左闭右开（最易理解）
    int result = 0; // 最终结果（根据场景初始化：比如最小窗口初始为Integer.MAX_VALUE）
    
    // 3. 右指针扩张窗口，遍历整个输入
    while (right < s.length()) { // 数组场景：nums.length
        // 3.1 取出当前元素，加入窗口
        char cur = s.charAt(right);
        window.put(cur, window.getOrDefault(cur, 0) + 1);
        // windowSum += nums[right]; // 数组和场景
        
        // 3.2 右指针右移，扩大窗口
        right++;
        
        // 4. 收缩左指针：当窗口满足「收缩条件」时，持续收缩
        while (需要收缩的条件) { // 核心：根据题目调整这个条件
            // 4.1 （关键）更新结果：只有收缩时才可能找到最优解（比如最小窗口）
            result = Math.min(result, right - left); // 示例：更新最小窗口长度
            
            // 4.2 取出左指针元素，移出窗口
            char leftChar = s.charAt(left);
            window.put(leftChar, window.get(leftChar) - 1);
            if (window.get(leftChar) == 0) {
                window.remove(leftChar); // 可选：减少冗余
            }
            // windowSum -= nums[left]; // 数组和场景
            
            // 4.3 左指针右移，收缩窗口
            left++;
        }
        
        // 可选：扩张时更新结果（比如最大窗口）
        // result = Math.max(result, right - left);
    }
    
    // 5. 结果兜底处理（比如未找到时返回0）
    return result == Integer.MAX_VALUE ? 0 : result;
}
```
