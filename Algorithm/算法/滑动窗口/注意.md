关键技巧（面试避坑）
窗口区间选择：推荐用「左闭右开 [left, right)」，这样 right - left 直接等于窗口长度，无需 + 1/-1，减少边界错误。
收缩条件设计：
最小窗口：满足条件时收缩（while 循环），收缩时更新结果。
最大窗口：不满足条件时收缩（while 循环），扩张后更新结果。
数据结构选择：
字母类问题：用 int[26] 替代 HashMap，效率更高（s.charAt(i) - 'a' 映射索引）。
数字和问题：直接用 int 变量记录窗口和，无需容器。
边界兜底：
最小窗口初始值设为 Integer.MAX_VALUE，最后判断是否更新，未更新则返回 0。
最大窗口初始值设为 0，无需兜底（至少有一个元素）。
总结
定长窗口：核心是「窗口长度达标后计算结果 + 收缩左指针」，逻辑简单，无内层 while 循环。
变长窗口：核心是「外层扩张右指针，内层 while 收缩左指针」，根据「最小 / 最大窗口」调整结果更新时机。
通用模板：记住「初始化→扩张→收缩→更新结果」四步，只需调整「收缩条件」和「结果更新逻辑」，就能解所有滑动窗口题。
建议先刷这 3 道经典题巩固模板：LeetCode 209（最小窗口和）、3（最大无重复子串）、643（定长窗口和），刷完后所有滑动窗口题都能套用模板解决。
