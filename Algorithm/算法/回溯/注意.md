核心本质
回溯算法是DFS 的一种特殊形式，核心是「尝试 - 回退 - 再尝试」，解决「找所有可能解」的问题（比如排列、组合、子集、路径），时间复杂度通常是 O (n!) 或 O (2ⁿ)，但通过剪枝能大幅优化。
回溯的核心流程：

选择当前节点 → 递归探索下一层 → 撤销选择（回退）→ 尝试下一个选择

关键剪枝技巧（面试提分核心）
提前终止：比如组合和超过 target 时，直接 return，无需递归下一层。
重复元素去重：先排序，再跳过i > start && nums\[i] == nums\[i-1]的元素（避免重复解）。
剩余元素不足剪枝：比如组合需要选 k 个元素，当前路径长度为path.size()，则 i 最大为n - (k - path.size()) + 1，超出的话直接终止循环。
已使用标记剪枝：排列问题中用used数组标记已选元素，避免重复选同一个元素。
总结
回溯模板核心：「终止条件→遍历选择列表→剪枝→做出选择→递归→撤销选择」六步。
场景适配：
组合 / 子集：start从 i+1 开始，无需used数组，靠start避免重复。
排列：start从 0 开始，必须用used数组标记已选元素。
带重复元素：先排序，再通过i > start && nums\[i] == nums\[i-1]剪枝。
剪枝是回溯题的核心优化手段，面试中写出剪枝逻辑能大幅提升通过率。
