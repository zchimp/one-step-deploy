# 不同方案
**中小规模 k8s 集群（节点数≤30，通用业务 / 中小数据库）：Longhorn + MinIO**  
理由：Longhorn 部署运维极简（一键部署），满足块存储需求；MinIO 原生云原生，满足对象存储需求，两者均开源、轻量，无需专业存储运维能力；

**中大规模 k8s 集群（节点数≥50，混合业务 / 核心数据库 / 大数据）：Ceph（RBD+CephFS+S3）**  
理由：三合一存储，一套集群满足块 / 文件 / 对象所有需求，社区成熟、生态丰富，支持大规模横向扩展，适合企业级核心业务，可通过 Rook 实现 k8s 自动化运维；

**大数据 / AI 训练场景：JuiceFS（文件存储） + MinIO（对象存储）**  
理由：JuiceFS 适配海量数据和多节点共享，MinIO 提供高性能对象存储作为数据湖底层，两者结合满足大数据处理和 AI 训练的海量数据存取需求；

**边缘 k8s 集群 / 轻量部署：OpenEBS Local PV + MinIO**  
理由：OpenEBS 实现本地存储分布式化，适配边缘节点资源有限的特点；MinIO 轻量可单机 / 分布式部署，满足边缘场景的对象存储需求；

**企业级商业需求（金融 / 电商，要求高可用 + 专业售后）：Portworx + 商业对象存储（如 AWS S3 / 阿里云 OSS）**  
理由：Portworx 提供企业级容灾、数据库优化、专业售后，云厂商对象存储提供无限扩展的非结构化数据存储能力。

# MinIO(商业化，开源版已经进入维护模式，不再更新)
## 部署 MinIO Operator
Operator 用于自动化管理 MinIO 集群生命周期，先部署 Operator 组件：
```
kubectl apply -f https://github.com/minio/operator/releases/download/v5.0.11/minio-operator.yaml
```
查看 Operator Pod 状态：
```
kubectl get pods -n minio-operator
```
## 创建 MinIO 租户（分布式集群核心）
### 创建租户凭证 Secret（存储访问密钥MINIO_ROOT_USER和MINIO_ROOT_PASSWORD），YAML：minio-secret.yaml
```
apiVersion: v1
kind: Secret
metadata:
  name: minio-tenant-secret
  namespace: minio-operator
type: Opaque
data:
  # 密钥需base64编码，示例：echo -n "admin123456" | base64
  MINIO_ROOT_USER: YWRtaW4xMjM0NTY=
  MINIO_ROOT_PASSWORD: YWRtaW4xMjM0NTZfcGFzc3dvcmQ=
```
### 创建 MinIO 租户 YAML：minio-tenant.yaml（4 节点分布式集群，冗余高可用）
```
apiVersion: minio.min.io/v2
kind: Tenant
metadata:
  name: minio-distributed
  namespace: minio-operator
spec:
  # 集群节点数
  pools:
  - servers: 4
    volumesPerServer: 2
    volumeClaimTemplate:
      spec:
        storageClassName: longhorn
        accessModes:
          - ReadWriteOnce
        resources:
          requests:
            storage: 10Gi
  # 关联密钥Secret
  credsSecret:
    name: minio-tenant-secret
  # 服务暴露方式
  service:
    type: NodePort
```
### 部署集群
```
kubectl apply -f minio-secret.yaml
kubectl apply -f minio-tenant.yaml
```
## 验证 MinIO 集群
### 查看租户 Pod 状态，等待所有 MinIO Pod 为Running
```
kubectl get pods -n minio-operator -l v1.min.io/tenant=minio-distributed -w
```
### 获取 MinIO 服务访问地址
```
kubectl get svc -n minio-operator minio-distributed-hl
```
记录NODEPORT端口，访问地址为http://<集群任意节点IP>:<NodePort端口>。
### S3 API 兼容性验证（使用 MinIO 客户端mc）
```
# 本地安装mc后，添加MinIO服务别名
mc alias set myminio http://<节点IP>:<端口> admin123456 admin123456_password
# 创建测试存储桶
mc mb myminio/test-bucket
# 上传文件验证
echo "test minio s3" > test-file.txt
mc cp test-file.txt myminio/test-bucket
# 列出存储桶文件
mc ls myminio/test-bucket
```
