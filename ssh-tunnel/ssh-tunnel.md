# 配置免密登陆
## 生成本地客户端的公钥和私钥
```
ssh-keygen
```
执行完毕后会在用户目录.ssh文件夹下创建公私钥：id_rsa （私钥）和id_rsa.pub (公钥)
## 上传公钥到远程服务器
```
ssh-copy-id -i ~/.ssh/id_rsa.pub root@192.168.1.2
```
### 测试登陆
```
ssh root@192.168.1.2
```

# ssh建立端口转发
最近在学习k8s相关的内容，但是云服务器就两台，一台性能还挺差的，无法搭建集群，所以就想在本机上玩一下，通过内网穿透的方式通过云主机的ip登陆本地的机器。

## 修改ssh的配置文件
修改 /etc/ssh/sshd_config
```
GatewayPorts yes
```
如果没有配置这一步的话，端口只能在本地访问，netstat可能看到端口只监听127.0.0.1的地址。

## 建立ssh隧道配置转发
执行格式：
ssh -R 18080:localhost:8080 HostB  
```
ssh -fN -R "[::]:9443:127.0.0.1:443" root@192.168.37.4
```
-fN 参数表示后台执行并且不执行命令  
-R 参数是执行远程转发
 [::] 匹配远程主机上的所有地址
 9443 远程主机上监听的端口  
 127.0.0.1:443 表示转发到本地的443端口  
 root@192.168.37.2 表示用root账户登录远程主机  
 
 执行完之后还需要输入密码，配置就成功了。当访问192.168.37.4:9443时，请求会转发到本机的443端口上。
